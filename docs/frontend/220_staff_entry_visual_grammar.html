Quiet_Internal_Control

Visual grammar for staff entry surfaces

Quiet internal control uses restrained neutral structure, one strong action accent, and explicit status labels. The goal is high-trust legibility for repeated staff work, not presentation density.

Tokens

Background

Muted neutral field for long sessions and high zoom.

Action accent

Reserved for the dominant next-safe action and active route state.

Blocking tone

Always paired with text labels such as Blocked or Held.

Layout

LayerRule
Mastheadintroduce shell family, scenario posture, and high-level route navigation
North-star ribbon5 or fewer concise posture metrics, never a dashboard wall
Main route bandone dominant pane and one secondary pane on desktop, one stacked flow on narrow widths
Continuity ledgerexplicit continuity metadata beside content rather than hidden state

Components

RecommendedQueueCard

Owns the dominant action and breach framing.

InterruptDigestStack

Unifies approvals, callbacks, escalations, and dependencies.

SupportDeskEntryPanel

Frames support as an inbox entry, not a detached dashboard widget.

BottleneckRadarLite

Ranks pressure visually without chart wallpaper.

SupportInboxViewSwitcher

Tabs between governed inbox slices in the same shell.

BlockingDependencyBanner

Only appears when action authority changes materially.

States

Quiet

Sparse operations entry and compact queue launch.

Busy

Higher counts and pressure, but the same action hierarchy.

Blocking

Dependency truth replaces launch enthusiasm.
