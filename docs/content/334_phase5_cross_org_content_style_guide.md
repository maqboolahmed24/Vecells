# 334 Phase 5 Cross-Org Content Style Guide

## Core rule

Equivalent truths use equivalent phrases across patient, hub, practice, and recovery surfaces.
Do not invent softer aliases for convenience.

## Canonical phrase family

### `Appointment confirmed`

- Use only when the current patient-facing reassurance is lawful for the current generation.
- This phrase does not imply the origin practice has acknowledged the same generation.
- Do not replace with `all confirmed`, `fully confirmed`, or `everyone informed`.

### `Practice informed`

- Use only when the current operational notice has been sent to the origin practice.
- Keep it operational and secondary to patient reassurance.
- Do not collapse this into `confirmed with practice`.

### `Practice acknowledged`

- Use only when receipt for the current generation has been recorded.
- Keep it distinct from both patient reassurance and dispatch.
- Do not rewrite as `confirmed` on its own.

### `Manage live`

- Use when the current capability tuple allows mutation from the route.
- Keep it tied to capability truth, not to reminder success or patient calmness.

### `Provider pending`

- Use when the requested change is visible but provider settlement has not yet completed.
- Do not soften this to `confirmed soon`.

### `Callback fallback`

- Use for the separate bounded fallback path.
- Do not relabel callback fallback as a reminder or delivery success.

## Artifact and placeholder language

### Summary-first framing

- Preferred title pattern: `Summary-first ... artifact`.
- Explain the current preview ceiling before exposing richer actions.
- Richer actions are secondary and must remain tied to the same route anchor.

### Governed placeholders

- State why detail is reduced.
- Keep the current visible summary and operational explanation in the same block.
- Do not leave empty tables, blank drawers, or vague copy such as `Unavailable right now`.

### Return receipts

- Use explicit return wording such as `Return to current summary or receipt` when the exact anchor label is not yet shown.
- Name the current return target directly: `Appointment summary`, `Network manage summary`, or `Settlement receipt`.
- Keep the summary sentence calm and specific: what action armed, what stayed bounded, and that the anchor was preserved.

## Copy do / do not

- Do: `Appointment confirmed`
- Do: `Practice informed`
- Do: `Practice acknowledged`
- Do: `Why richer artifact detail is held back`
- Do not: `Everything confirmed`
- Do not: `Practice confirmed`
- Do not: `Export ready` without naming what stayed bounded
- Do not: `Preview unavailable` without explaining the ceiling
