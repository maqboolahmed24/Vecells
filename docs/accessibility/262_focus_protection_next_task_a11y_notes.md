# 262 Focus Protection And Next Task Accessibility Notes

- Keep `WorkspaceProtectionStrip`, `BufferedQueueChangeTray`, `CompletionContinuityStage`, `NextTaskPostureCard`, and `DepartureReturnStub` inside the normal heading flow. They are operational state, not decorative chrome.
- `data-auto-advance="forbidden"` is a user-protection law. Keyboard users must never lose their current focus target because next-task posture changes.
- The buffered tray stays in the tab order and does not announce as a toast.
- Protected or stale-recoverable recovery preserves the prior structure so screen-reader users keep orientation.
- Reduced motion must keep all meaning through copy, outline, and state attributes rather than relying on movement.
- Blocked or stale next-task posture keeps the CTA disabled with visible reasons in the DOM.
