# 265 Self-Care Admin Views Accessibility Notes

- Keep the worklist, boundary digest, dominant stage, dependency lane, and patient expectation preview inside predictable landmarks and heading order.
- Keyboard flow must reach the worklist first, then the dominant self-care or admin stage, then blocker and expectation context.
- The self-care note textarea must keep a visible focus ring and remain readable when frozen in stale-recoverable posture.
- The admin subtype selector uses button semantics inside a listbox-style grouping so selection state remains screen-reader legible.
- Reduced motion must preserve meaning with outline, tone, and density changes rather than spatial travel.
- Patient expectation preview needs clear wording that distinguishes `full`, `summary_safe`, and `placeholder_safe` delivery modes without jargon.
- Boundary drift and reopen posture must remain announced through visible copy, not color alone.
