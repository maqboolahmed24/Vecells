## Semantic Parity Rule

Phone provenance may add a provenance chip or a helpful explanatory note, but it may not change canonical field meaning, duplicate classification, safety entry criteria, patient-visible status family, or receipt semantics.

## Execution Note

Mock-now verification is driven by local fixtures, command-api suites, and Playwright coverage for the browser lab. Live-provider-later auth, telephony, and support correction paths must reuse the same semantic parity matrix and wrong-patient hold and release semantics rather than adding channel-specific request truth. Same-shell recovery preserves the selected anchor and suppresses PHI whenever identity hold is active.
