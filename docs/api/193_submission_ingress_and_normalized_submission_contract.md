# 193 Submission Ingress And Normalized Submission Contract

The convergence command is `TelephonyConvergenceCommandContract`, exposed through `telephony_convergence_command_submit`.

## Command Inputs

Required command fields include:

- `commandId`, `idempotencyKey`, `tenantId`, and `sourceLineageRef`.
- `ingressChannel`: `self_service_form`, `telephony_capture`, `secure_link_continuation`, or `support_assisted_capture`.
- `requestType`, narrative sources, structured answer sources, metadata refs, identity context, attachment refs, audio refs, contact preferences, source timestamp, dedupe refs, evidence readiness, capability ceiling, contact authority, and receipt seed.

The API accepts multiple evidence sources but normalizes only after freezing source precedence. For phone plus continuation, continuation-entered fields beat keypad, transcript, and spoken values. For support-assisted capture, support-entered values beat continuation, keypad, transcript, and spoken values.

## Durable Outputs

`SubmissionIngressRecord` is the canonical join between entry channel and intake. It includes the ingress channel, surface profile, capture authority, promotion intent, capability ceiling, contact authority class, readiness state, evidence refs, normalized submission ref, payload hash, and receipt/status keys.

`NormalizedSubmission` is generated by the shared normalizer. The convergence layer widens freeze input channels and audio refs but does not create a phone-specific normalized request model.

`ConvergenceDuplicateResolutionDecision` records the durable duplicate decision class. `same_request_attach` requires an explicit continuity witness and no material divergence. Missing channel calibration fails closed to review.

`ReceiptConsistencyKeyContract` serializes the patient-visible receipt/status grammar. The key grammar is independent of channel and is based on tenant, request type, semantic meaning hash, and lineage consistency seed.

## Routes

- `POST /internal/telephony/convergence/submit`
- `GET /internal/telephony/convergence/{convergenceOutcomeRef}`
- `GET /internal/telephony/convergence/{convergenceOutcomeRef}/receipt-status`
- `POST /internal/telephony/convergence/{convergenceOutcomeRef}/readiness-resume`
