# Session Governor Design

Task: `par_176_phase2_track_identity_build_session_governor_for_session_establishment_rotation_and_logout`

## Purpose

`SessionGovernor` is the only local Vecells authority allowed to create, rotate, reuse, downgrade, revoke, or terminate server-side sessions. NHS login success, auth-bridge callback settlement, evidence-vault refs, binding intents, patient-link decisions, and capability decisions are inputs. They are not a usable local session by themselves.

The implementation lives in `services/command-api/src/session-governor.ts` and exposes `createSessionGovernorApplication()`.

## Authoritative Objects

| Object | Rule |
| --- | --- |
| `SessionEstablishmentDecision` | Decides `create_fresh`, `rotate_existing`, `reuse_existing`, `deny`, or `bounded_recovery` with subject comparison, draft-claim disposition, return-intent disposition, writable-authority state, cookie action, CSRF action, and grant ceiling. |
| `LocalSession` | Server-side session record. Browser state carries only the secure HTTP-only session cookie and CSRF material required by policy. |
| `SessionTerminationSettlement` | Exact settlement for logout, idle timeout, absolute timeout, revocation, downgrade, forced termination, subject conflict, and stale return. Missing cookies are not treated as complete truth. |
| `SessionProjectionMaterialization` | Least-privilege posture projection for `active`, `step_up_required`, `restricted`, `recovery_only`, `expired_idle`, `expired_absolute`, `revoked`, or `terminated`. |

## Establishment Decisions

| Input condition | Decision | Session action |
| --- | --- | --- |
| Settled subject, binding, patient-link, and capability refs with no existing session | `create_fresh` | Issue new session id, CSRF secret, and `sessionEpochRef`. |
| Existing same-subject session with unchanged binding and capability refs | `reuse_existing` | Touch last-seen and idle timeout; do not rotate cookie or CSRF. |
| Existing same-subject session with changed binding, privilege, or capability refs | `rotate_existing` | Mark old session `rotated`; issue new session id, CSRF secret, and `sessionEpochRef`. |
| Auth success has only binding/capability intents | `bounded_recovery` | No local writable session; project recovery-only posture. |
| Existing cookie belongs to a different subject | `bounded_recovery` plus `SessionTerminationSettlement(subject_conflict)` | Tear down the previous subject session and refuse writable continuation. |

This preserves the distinction between authentication proof, patient linkage, identity binding, capability, and local session state.

## Timeout and Re-auth Policy

The governor uses the Phase 2 web posture of 30 minutes idle timeout and 12 hours absolute continuous usage. These values are serialized as `SESSION_IDLE_TIMEOUT_SECONDS = 1800` and `SESSION_ABSOLUTE_TIMEOUT_SECONDS = 43200`.

Timeouts are not inferred from cookie absence. The governor emits an exact `SessionTerminationSettlement` for `idle_timeout` or `absolute_timeout`, clears the cookie, and materializes `expired_idle` or `expired_absolute`.

## Cookies and CSRF

Session cookies are issued as `__Host-vecells.sid` with `Path=/`, `HttpOnly`, `Secure`, `SameSite=Lax`, and `Max-Age=43200`. The raw session identifier is never stored in operational rows; the repository stores `cookieDigest`.

CSRF secrets are generated by the server and stored as `csrfSecretDigest`. The session id, CSRF secret, and `sessionEpochRef` rotate whenever a same-subject session receives a different binding version, identity binding ref, capability decision ref, or writable-authority state.

## Ports and Integration

The service exposes `settleSessionEstablishment()` for the auth bridge port. That method deliberately returns `bounded_recovery` when auth success only includes binding and capability intents, because a live writable session requires settled identity-binding, patient-link, and capability refs.

Later tasks can plug in their settled refs through `establishSession()` without changing the auth bridge or local session semantics.

## Gap Closures

| Gap | Closure |
| --- | --- |
| `PARALLEL_INTERFACE_GAP_PHASE2_SESSION_ESTABLISHMENT_DECISION_V1` | Auth success is input only; the establishment engine decides local session posture. |
| `PARALLEL_INTERFACE_GAP_PHASE2_SESSION_COOKIE_CSRF_ROTATION_V1` | Session id, CSRF secret, and epoch rotate on binding, privilege, and capability changes. |
| `PARALLEL_INTERFACE_GAP_PHASE2_SESSION_TIMEOUT_SETTLEMENT_V1` | Idle, absolute, logout, downgrade, revocation, and subject-switch outcomes are settlement records. |
| `PARALLEL_INTERFACE_GAP_PHASE2_SESSION_AUTH_BRIDGE_PORT_V1` | Auth bridge can call a bounded session-governor port without owning session logic. |
| `PARALLEL_INTERFACE_GAP_PHASE2_SESSION_PROJECTION_POSTURE_V1` | Session posture is materialized from server-side authority, not browser-local state. |
