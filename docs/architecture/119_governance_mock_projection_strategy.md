# 119 Governance Mock Projection Strategy

## Mock now

The Phase 0 governance seed uses truthful mock projections instead of placeholder admin forms.

- scope tuples remain explicit: tenant, organisation, environment, purpose-of-use, active review object, and freeze disposition
- drafts never appear production-approved unless the selected route is already the approval or release lane
- publication drift, watch drift, and scope drift suppress controls in place instead of silently refreshing the current package
- evidence bundles, release tuples, and access previews stay summary-first and same-shell

## Seeded scenario set

- live config bundle review with `ChangeEnvelope` plus `ImpactPreview`
- read-only compliance bundle with explicit export suppression
- blocked access review caused by scope drift
- recovery-only release watch caused by freeze or publication conflict
- calm tenant matrix posture with aligned inherited, live, and draft values
- template review frozen by publication parity drift

## Actual production strategy later

- preserve the same route family and shell topology
- bind real compile, promotion, publication, access, records, and evidence systems into the existing surfaces instead of replacing them
- keep `ScopeRibbon`, `GovernanceFreezeDisposition`, `GovernanceReturnIntentToken`, and the same selected-anchor acknowledgement rule as the production control law
- deepen legal, export, simulation, and external evidence assembly only behind the same continuity frame

## Anti-patterns explicitly blocked

- detached CRUD pages for tenant config, access review, template editing, or release watch
- stale scope or freeze state leaving controls apparently writable
- evidence panels that look authoritative while publication parity or watch posture has already drifted
- approval flows that discard the current baseline snapshot or review package hash
