# 396 External Reference Notes

Reviewed on 2026-04-27. Local blueprints remain the source of truth; these references only shape current operational expectations.

| Source | Borrowed | Rejected or bounded |
| --- | --- | --- |
| NHS England Digital, NHS App web integration: https://digital.nhs.uk/services/nhs-app/how-to-integrate-with-the-nhs-app/nhs-app-web-integration | Sandpit comes before AOS; SCAL is a reviewed evidence package; limited release and post-live monthly data follow after readiness. | The repository does not claim NHS sign-off, SCAL acceptance, connection-agreement completion, or go-live approval. |
| NHS App developer guidance, web integration overview: https://nhsconnect.github.io/nhsapp-developer-documentation/web-integration-overview/ | NHS App web integration uses a tailored webview, jump-off navigation, custom user agent, hidden supplier header, and JS API actions. | User-agent detection alone is not treated as trusted authority; local signed context and release tuple checks still govern. |
| NHS App developer guidance, web integration guidance: https://nhsconnect.github.io/nhsapp-developer-documentation/web-integration-guidance/ | `assertedLoginIdentity` can arrive in the URL and must be handled as sensitive; site-link values vary by environment. | Example JWTs and environment IDs are not committed as real credentials or session material. |
| NHS App JS API v2 specification: https://nhsconnect.github.io/nhsapp-developer-documentation/js-v2-api-specification/ | v2 is the current JS API surface; functions include app detection, navigation, calendar, external browser, and byte download. | The onboarding pack validates capability posture but does not call live NHS App JS endpoints outside approved environments. |
| NHS digital service manual accessibility: https://service-manual.nhs.uk/accessibility | NHS services must meet WCAG 2.2 expectations and make accessibility evidence ready before progression. | Automated checks are evidence support only; they do not replace manual/accessibility audit evidence where required. |
| NHS digital service manual accessibility testing: https://service-manual.nhs.uk/accessibility/testing | Accessibility testing must combine automated checks, manual checks, and assistive-technology review; keyboard and heading structure remain explicit. | Screenshots alone are not evidence when semantic or tuple validation fails. |
| Playwright browser context isolation: https://playwright.dev/docs/browser-contexts | Browser contexts provide clean-slate isolated state for deterministic operator-flow automation. | Persistent signed-in browser state is not reused in onboarding scripts. |
| Playwright tracing and trace viewer: https://playwright.dev/docs/api/class-tracing and https://playwright.dev/docs/trace-viewer | Traces can capture snapshots/screenshots and should be exported intentionally; the browserContext tracing API is valid outside the test runner. | Trace export is allowed only from redaction-safe local pages in this repository package. |
